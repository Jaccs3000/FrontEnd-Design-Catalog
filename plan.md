# Frontend Design Catalog

## Plataforma Educativa y Profesional para Diseño Frontend Moderno

---

# 1. Visión del Proyecto

Crear una plataforma web interactiva enfocada en:

* aprender diseño frontend moderno
* explorar componentes UI
* entender sistemas de diseño
* visualizar patrones UX/UI
* comparar librerías y frameworks
* estudiar motion design
* servir como referencia para proyectos empresariales
* ayudar a personas sin experiencia avanzada en frontend

La plataforma NO será únicamente una galería de componentes.

Será un:

* laboratorio visual
* playground frontend
* sistema educativo UI/UX
* catálogo profesional
* referencia visual y técnica
* sistema base para futuros proyectos web

---

# 2. Objetivos del Proyecto

## Objetivos Educativos

La plataforma debe ayudar a:

* aprender nombres técnicos UI
* entender cuándo usar cada componente
* desarrollar criterio visual
* estudiar jerarquía visual
* comprender responsive design
* analizar buenas prácticas UX/UI
* estudiar motion design moderno

---

## Objetivos Técnicos

La plataforma debe:

* mostrar componentes reales
* permitir interacción visual
* documentar frameworks y librerías
* mostrar estados y variantes
* enseñar accesibilidad
* servir como referencia reusable
* facilitar prototipado rápido

# 4. Filosofía del Proyecto

## El proyecto NO debe:

* parecer documentación aburrida
* enfocarse solo en código
* usar diseño anticuado
* abusar de efectos visuales
* usar demasiados colores
* mostrar componentes aislados sin contexto

---

## El proyecto SÍ debe:

* verse moderno y premium
* ser visualmente limpio
* enseñar mediante ejemplos reales
* permitir exploración interactiva
* explicar decisiones UX/UI
* mostrar patrones profesionales

# 12. Estructura de Cada Componente

Cada componente debe contener:

---

## Nombre técnico

Ejemplo:

* Modal
* Tooltip
* Accordion

---

## Descripción breve

Explicar:

* qué es
* qué problema resuelve
* propósito principal

---

## Cuándo usarlo

Explicar:

* casos ideales
* patrones recomendados

---

## Cuándo NO usarlo

Explicar:

* malas prácticas
* problemas UX comunes

---

## Librería utilizada

Ejemplo:

* shadcn/ui
* Tailwind CSS
* Framer Motion

---

## Framework requerido

Ejemplo:

* React
* Next.js

---

## Nivel de dificultad

* Básico
* Intermedio
* Avanzado

---

## Responsive Behavior

Explicar:

* comportamiento móvil
* comportamiento desktop

---

## Accesibilidad

Explicar:

* keyboard navigation
* focus states
* screen readers
* aria support

---

## Estados del componente

Mostrar:

* default
* hover
* focus
* active
* disabled
* loading
* error

---

## Variantes

Ejemplo:

* primary
* secondary
* outline
* ghost

---

## Preview interactivo

El usuario debe poder:

* interactuar
* probar estados
* visualizar animaciones

---

## Código fuente (opcional)

Mostrar:

* implementación básica
* librería utilizada

---

## Análisis visual

Explicar:

* por qué el diseño funciona
* jerarquía visual
* spacing
* contraste
* alineación

# 14. Visual Quality Rules

## La UI debe evitar:

* exceso de colores
* sombras exageradas
* demasiados gradientes
* animaciones agresivas
* texto centrado innecesariamente
* exceso de glassmorphism

---

## La UI debe priorizar:

* whitespace
* legibilidad
* consistencia
* jerarquía visual
* simplicidad
* claridad visual

---
